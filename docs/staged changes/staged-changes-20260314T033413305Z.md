# Staged Changes Documentation

## Summary

These staged changes fix the `Đội bóng` data visibility issue after database reset and harden teams page handling.

Main outcomes:

- Resetting DB with `resources/initialize.sql` now restores seeded team data correctly.
- `/teams` no longer falls into `404` when there are zero teams.
- Team model mapping logic is refactored to reduce duplication and preserve correct module export behavior.

## Changed Files

- `src/resources/initialize.sql`
- `src/controllers/teams.c.js`
- `src/models/team.m.js`

## Detailed Changes

### 1) `src/resources/initialize.sql`

Reordered seed data blocks to satisfy foreign-key dependencies during import.

New effective order:

1. `users`
2. `tournaments`
3. `teams`
4. `players`
5. `teams_statistics`
6. `matches`
7. `match_events`

Why:

- Previously, `teams`/`players` were inserted before referenced parent rows (`users`, `tournaments`, `teams`), causing FK violations.
- `psql -f` still returned success exit code while skipping failed inserts, resulting in missing teams.

### 2) `src/controllers/teams.c.js`

Updated teams pagination handling in `getTeams`:

- Parse page value with `Number.parseInt(req.query.page, 10) || 1`.
- Compute `maxPage` with safe fallback (`1` when there are no teams).
- Return `next()` only for truly invalid page values (`page < 1 || page > maxPage`).

Why:

- Prevents `/teams` from rendering `404` when team count is zero and requested page is default page 1.

### 3) `src/models/team.m.js`

Refactor and stability updates:

- Added `buildTeamsWithPlayerCounts(teams)` helper to centralize model mapping and player count enrichment.
- Reused helper in `getAllTeams`, `getAllCurrentTeams`, and `getAllActiveTeams`.
- Preserved valid class/module structure by exporting with `module.exports = TeamModel;`.

Why:

- Reduces duplicate code paths.
- Avoids runtime issues from class export/reference scope mistakes.
